## 二、Linux 内核中高频使用的数据结构

作为内核开发者，这些数据结构必须熟悉：

| 数据结构 | 内核中的使用场景 | 内核源码位置 |
|----------|------------------|-------------|
| **双向链表 (list_head)** | 几乎所有子系统都用到 | `include/linux/list.h` |
| **红黑树 (rbtree)** | CFS调度器、VMA管理、ext4文件系统 | `include/linux/rbtree.h` |
| **哈希表 (hlist)** | 页缓存、dentry缓存、inode缓存 | `include/linux/list.h` |
| **基数树 (radix tree)** | 页缓存索引 | `lib/radix-tree.c` |
| **位图 (bitmap)** | 内存管理 Buddy System、CPU掩码 | `include/linux/bitmap.h` |
| **队列 (kfifo)** | 内核日志缓冲区、中断处理 | `include/linux/kfifo.h` |
| **红黑树+链表(XArray)** | 新版页缓存 | `lib/xarray.c` |
| **跳表** | 定时器管理 (timer wheel) | `kernel/time/timer.c` |
| **堆 (优先队列)** | 调度器 | `kernel/sched/` |
| **LRU链表** | 内存回收 | `mm/swap.c` |

## 三、内核开发者的算法学习重点

### 第一阶段：基本数据结构（必会）

```
链表（单向/双向/循环） → 栈 → 队列 → 哈希表 → 二叉树 → 堆
```

**练习**：用 C 实现一个泛型双向链表（模仿内核的 list_head）

### 第二阶段：查找与排序（必会）

```
线性查找 → 二分查找 → 快速排序 → 归并排序 → 堆排序
```

**练习**：实现 qsort 和 bsearch，理解时间复杂度

### 第三阶段：平衡树（内核常用）

```
二叉搜索树 → AVL树 → 红黑树 → B/B+树
```

**重点**：红黑树 — CFS 调度器、VMA、ext4 都依赖它

### 第四阶段：哈希与位图（内核高频）

```
哈希函数 → 哈希冲突解决 → 位图操作 → 布隆过滤器
```

### 第五阶段：图与字符串（扩展）

```
BFS/DFS → 最短路径 → Trie前缀树 → KMP字符串匹配
```

## 五、零基础数据结构学习顺序

```
第1-2周：数组 + 链表（最重要的是看懂指针操作）
第3周：栈 + 队列
第4周：哈希表
第5-6周：二叉树 + 二叉搜索树
第7周：堆 + 堆排序
第8周：红黑树（内核重点，比AVL树更常用）
第9周：图（BFS + DFS）
第10-12周：刷 LeetCode 巩固（重点关注链表、树、哈希表相关题目）
```

## 关键提醒

1. **用 C 实现每种数据结构** — 不要用 Python/Java 偷懒
2. **理解时间复杂度** — 知道什么时候用什么结构
3. **读内核中的实现** — `list.h` 的双向链表实现非常巧妙
4. **红黑树是内核面试高频点** — 至少要知道它的基本原理和性质
