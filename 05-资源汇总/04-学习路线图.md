# 五阶段详细学习路线图

> 从零基础到高通 Linux 内核工程师的完整路线，预计总时长 3-5 年。

---

## 阶段一：计算机基础（0-6个月）

### 目标
- 理解计算机的硬件组成和工作原理
- 掌握 C 语言编程（内核的母语）
- 理解操作系统基本概念
- 掌握最基本的算法和数据结构

### 学习模块

| 序号 | 模块 | 时间 | 状态 |
|------|------|------|------|
| 1.1 | 计算机组成原理 | 3-4周 | 进行中 |
| 1.2 | 操作系统基础 | 3-4周 | 待开始 |
| 1.3 | 数据结构与算法 | 4-6周 | 待开始 |
| 1.4 | C语言编程 | 6-8周 | 待开始 |

### 核心知识点

**计算机组成原理**
- 冯·诺依曼体系结构
- CPU工作过程（取指/译码/执行/写回）
- 存储器层次结构（寄存器/Cache/内存/外存）
- 总线系统（数据/地址/控制总线）
- 中断与异常

**操作系统基础**
- 进程与线程
- 内存管理基础（虚拟内存、分页）
- 文件系统概念
- 系统调用
- I/O 管理

**数据结构与算法**
- 数组、链表、栈、队列
- 哈希表、树（二叉树、红黑树）
- 排序与查找算法
- 时间复杂度分析
- 内存池、位图（内核中大量使用）

**C语言编程**
- 指针与内存管理（malloc/free、栈vs堆）
- 结构体、联合体、枚举
- 函数指针
- 预处理器与宏
- 位运算（寄存器操作必须）
- volatile、const、static

### 实践项目
1. 用 C 实现一个简单的链表
2. 用 C 实现一个内存池分配器
3. 编译一个最简单的 Linux 内核
4. 写一个 Hello World 内核模块

---

## 阶段二：Linux 基础（6-12个月）

### 目标
- 熟练使用 Linux 作为日常操作系统
- 能编写 Shell 脚本和 Makefile
- 掌握 Linux 系统编程（进程/信号/管道/文件I/O）
- 理解 Linux 的启动流程

### 学习模块

| 序号 | 模块 | 时间 | 状态 |
|------|------|------|------|
| 2.1 | Linux系统使用 | 2-3周 | 待开始 |
| 2.2 | Linux命令行 | 2-3周 | 待开始 |
| 2.3 | Shell脚本编程 | 2-3周 | 待开始 |
| 2.4 | Linux系统编程 | 6-8周 | 待开始 |

### 核心知识点

**Linux系统使用**
- 文件系统层次结构（FHS标准）
- 用户与权限管理（UID/GID、chmod、chown）
- 包管理器（apt、yum）
- systemd 服务管理

**Linux命令行**
- 文件操作（ls, cp, mv, rm, find, grep）
- 管道与重定向（|, >, <, >>）
- 进程管理（ps, top, kill, nice）
- 网络命令（ping, netstat, ip, ssh）

**Shell脚本编程**
- Bash 语法（变量/循环/条件/函数）
- sed、awk 文本处理
- crontab 定时任务
- 自动化脚本实战

**Linux系统编程**
- 文件 I/O（open/read/write/lseek/close vs fopen/fread/fwrite/fclose）
- 进程管理（fork/exec/wait/exit）
- 信号处理（signal/sigaction）
- 管道与FIFO（pipe/mkfifo）
- 共享内存与信号量（shmget/semget）
- 套接字编程（socket/bind/listen/accept/connect）

### 实践项目
1. 写一个简单的 Shell（支持管道和重定向）
2. 用 C 实现一个 TCP echo 服务器
3. 写一个进程守护进程（daemon）
4. 用 Shell 脚本实现一个系统监控工具

## 阶段三：Linux 内核基础（12-24个月）

### 目标
- 深入理解 Linux 内核的核心子系统
- 能编写和调试字符设备驱动
- 会使用 ftrace/perf/eBPF 追踪内核行为
- 能阅读和理解内核关键路径源码

### 学习模块

| 序号 | 模块 | 时间 | 状态 |
|------|------|------|------|
| 3.1 | 内核架构与启动过程 | 3-4周 | 待开始 |
| 3.2 | 进程管理与调度 | 4-6周 | 待开始 |
| 3.3 | 内存管理 | 6-8周 | 待开始 |
| 3.4 | 设备驱动模型 | 6-8周 | 待开始 |
| 3.5 | 文件系统（VFS） | 4-6周 | 待开始 |
| 3.6 | 网络协议栈 | 4-6周 | 待开始 |

### 核心知识点

**内核架构与启动**
- 内核源码目录结构
- 启动流程：Bootloader → setup_arch → start_kernel → rest_init
- 内核编译与配置（make menuconfig）
- 内核模块机制（insmod/rmmod/modprobe）
- 设备树基础（DTS/DTB）

**进程管理与调度**
- task_struct 结构体
- 进程状态（TASK_RUNNING、TASK_INTERRUPTIBLE等）
- CFS 完全公平调度器
- 实时调度策略（FIFO/RR）
- 抢占与上下文切换
- 内核线程（kthread）

**内存管理**
- 物理内存管理（Buddy System / Slab分配器）
- 虚拟内存管理（页表、VMA）
- kmalloc/vmalloc/malloc 的区别
- 缺页异常处理（page fault）
- 内存回收（kswapd、OOM Killer）
- Page Cache 与 Buffer Cache
- CMA（Contiguous Memory Allocator）— 高通平台重要
- SMMU（System Memory Management Unit）— 高通平台核心

**设备驱动模型**
- Platform Device / Platform Driver 模型
- 设备树与驱动的匹配（compatible 属性）
- 字符设备驱动（cdev / file_operations）
- 中断处理（request_irq / 顶半部与底半部）
- 并发与同步（spinlock / mutex / semaphore）
- DMA（Direct Memory Access）
- I2C/SPI/GPIO 子系统

**文件系统（VFS）**
- VFS 层（super_block / inode / dentry / file）
- 文件操作（open/read/write 的内核实现路径）
- procfs / sysfs / debugfs
- 块设备层与 I/O 调度
- ext4、F2FS 等文件系统

**网络协议栈**
- Socket 层与协议族
- TCP/IP 实现概览
- NAPI（New API）收包机制
- netfilter 与 iptables
- eBPF 与 XDP

### 实践项目
1. 用 QEMU 运行自己编译的内核
2. 写一个完整的字符设备驱动（支持 read/write/ioctl）
3. 用 platform driver 模型写一个 I2C 设备驱动
4. 用 ftrace 追踪一次 open() 系统调用的完整路径
5. 写一个 procfs 接口来暴露内核数据到用户空间
6. 创建一个 debugfs 节点用于调试

## 阶段四：ARM64 + 高通平台（24-36个月）

### 目标
- 掌握 ARM64 架构基础知识
- 理解高通骁龙 SoC 的硬件架构
- 能开发和调试高通平台设备驱动
- 熟悉 Camera/GPU/Modem 驱动框架

### 学习模块

| 序号 | 模块 | 时间 | 状态 |
|------|------|------|------|
| 4.1 | ARM64架构基础 | 4-6周 | 待开始 |
| 4.2 | 高通骁龙平台架构 | 6-8周 | 待开始 |
| 4.3 | 高通设备树(DTS) | 3-4周 | 待开始 |
| 4.4 | 高通驱动框架(时钟/电源/引脚) | 6-8周 | 待开始 |
| 4.5 | Camera/GPU/Modem 驱动 | 8-12周 | 待开始 |
| 4.6 | SMMU/DPD/Proxy 机制 | 6-8周 | 待开始 |
| 4.7 | 性能调优与调试 | 4-6周 | 待开始 |

### 核心知识点

**ARM64架构**
- 异常级别（EL0用户空间 ↔ EL1内核 ↔ EL2虚拟化 ↔ EL3安全）
- MMU 与页表格式（4级页表，48位虚拟地址）
- GIC 中断控制器（GICv3/GICv4 高通使用）
- SMMU — IOMMU 的 ARM 实现（高通平台核心）
- 缓存一致性（CCI/CCN互联）
- ARM TrustZone 安全扩展

**高通骁龙平台架构**
- CPU集群（Kryo核心 = ARM Cortex定制版）
- RPMh（资源电源管理）
- APCS（应用处理器通信子系统）
- QUP（Qualcomm Universal Peripheral）串行引擎
- IPA（IP Accelerator）网络加速
- CodeLinaro 内核分支（msm-5.x）

**高通设备树(DTS)**
- `arch/arm64/boot/dts/qcom/` 目录阅读
- `qcom,msm-id` / `qcom,board-id` 识别
- 定义：CPU频率表、Regulator、GPIO、Clock、中断映射
- 驱动如何通过 `compatible` 属性与设备树绑定

**高通驱动框架**
- `drivers/clk/qcom/` — GCC (Global Clock Controller)
- `drivers/pinctrl/qcom/` — TLMM (Top Level Mode Multiplexer) GPIO
- `drivers/soc/qcom/` — SoC 级驱动（RPM、SMEM、SMD等）
- `drivers/regulator/qcom/` — 电源管理
- `drivers/iommu/arm/arm-smmu/` — 高通使用的 ARM SMMU

**Camera/GPU/Modem 驱动**
- Camera: CamX/Chi-CDK 框架、V4L2、ISP Pipeline
- GPU: KGSL（旧）/ DRM MSM（新）、Adreno
- Modem: QMI协议、RPMSG/QRTR传输、Modem固件管理
- Hexagon DSP: QNN运行时、传感器框架

**SMMU / DPD / Proxy 机制**
- SMMU: 为设备提供地址翻译和内存保护
- DPD (Deep Power Down): 深度省电状态管理
- Proxy: 代理机制，处理跨电源域通信

**调试与调优**
- KGDB/JTAG 硬件调试
- QXDM Modem 日志分析
- perf/QPA 性能分析
- 内核崩溃转储（ramdump）分析

### 实践项目
1. 在 DragonBoard 845c 上编译运行高通内核
2. 编写一个 GPIO 设备树节点 + 驱动
3. 追踪 SMMU 的 page fault 处理流程
4. 修改设备树中的 Regulator 配置
5. 用 perf 分析内核函数的热点路径

## 阶段五：贡献与专家（36个月+）

### 目标
- 向 Linux 主线内核提交补丁
- 独立负责平台技术方案
- 主导新技术预研

### 核心能力
1. **上游贡献**：写高质量的 patch，通过 LKML 审核
2. **架构设计**：为新产品设计内核驱动架构
3. **跨子系统协作**：Camera + GPU + DSP + Modem
4. **技术规划**：跟踪主线内核变化，同步到高通分支
5. **性能优化**：系统级性能瓶颈分析

### 实践路径
1. 从修复小 bug 开始向上游提交 patch
2. 参与 linux-arm-msm 邮件列表讨论
3. 在内核社区建立个人声誉
4. 持续关注 LWN.net 的技术文章
