## 三、ARM SMMU — 你的工作核心

### SMMU 基础知识

```
           CPU                     Device (如 Camera/GPU)
            │                              │
            ▼                              ▼
        MMU (页表)                    SMMU (页表)
       虚拟→物理                      IOVA→物理
            │                              │
            └──────────┬───────────────────┘
                       ▼
                   物理内存
```

SMMU (System Memory Management Unit) 是 ARM 架构中的 IOMMU 实现。

**关键功能**：
1. **设备地址翻译**：将设备发出的 IOVA 转换为物理地址
2. **内存保护**：防止设备越界访问
3. **地址隔离**：不同设备看到不同的地址空间
4. **虚拟化支持**：SMMUv2 (arm-smmu) → SMMUv3

**你在研究的内容**：
- DPD (Deep Power Down)：深度省电模式下 SMMU 的状态保存/恢复
- Proxy 机制：通过代理处理跨电源域的 SMMU 访问

### SMMU 源码阅读路径

```
drivers/iommu/arm/arm-smmu/
  arm-smmu.c          — SMMUv2 驱动核心
  arm-smmu-qcom.c     — 高通特定的 SMMU 扩展
  arm-smmu-impl.c     — 厂商实现接口

drivers/iommu/arm/arm-smmu-v3/
  arm-smmu-v3.c       — SMMUv3 驱动

drivers/iommu/
  iommu.c             — IOMMU 通用框架
  iommu-debugfs.c     — debugfs 调试接口
  dma-iommu.c         — DMA 与 IOMMU 结合
  io-pgtable-arm.c    — ARM 页表实现
```

### SMMU 关键概念速查

| 概念 | 说明 |
|------|------|
| Stream ID (SID) | 每个设备有唯一 ID，用于 SMMU 识别 |
| Stream Matching | SMMU 根据 SID 匹配对应的上下文 |
| Context Bank | 每个设备上下文有独立的页表基址寄存器 |
| TLB | SMMU 内部的地址翻译缓存 |
| IOVA | I/O Virtual Address，设备看到的地址 |
| Page Table Walk | 遍历页表完成 IOVA → PA 翻译 |
| SMR | Stream Matching Register |
| S2CR | Stream-to-Context Register |
| Global Fault | SMMU 检测到的全局错误中断 |
| Context Fault | 特定上下文的事务错误 |

## 四、高通平台关键技术栈

### 1. 时钟管理 (Clock)

高通使用 GCC (Global Clock Controller) 管理 SoC 时钟。

```
drivers/clk/qcom/
  gcc-msm8998.c     — 示例：MSM8998 的 GCC 驱动
  clk-rcg.h          — Root Clock Generator
  clk-branch.h       — 分支时钟
```

**关键概念**：
- RCGR (Root Clock Generator Register)
- CBCR (Clock Branch Control Register)
- RCG → Branch 的时钟树结构

### 2. 电源管理 (RPM / RPMh)

```
drivers/soc/qcom/
  rpmh.c                  — 资源电源管理器(hardened)
  rpmh-rsc.c              — 资源状态协调器
  cmd-db.c                — 命令数据库
  rpmhpd.c                — RPMh 电源域
```

**关键概念**：
- RPMh：新一代电源管理器（SDM845+）
- RSC (Resource State Coordinator)：电源状态协调
- Sleep/Wake sets：休眠和唤醒寄存器组

### 3. 内存管理（高通特有）

- **CMA (Contiguous Memory Allocator)**：分配大块连续物理内存
- **ION → DMA-BUF**：高通内存分配器的演进
- **SMMU**：设备内存访问的保护和隔离

### 4. 远程处理器 (remoteproc) — DSP/Modem

```
drivers/remoteproc/
  qcom_q6v5_pas.c     — 启动 DSP/Modem 固件
  qcom_sysmon.c        — 子系统监控
  qcom_common.c        — 通用功能
  qcom_pil_info.c      — PIL (Peripheral Image Loader)
```

## 五、学习 Qualcomm 内核的推荐路径

### 第1步：搭建环境（1-2周）
```
1. 下载 CodeLinaro msm-5.15 内核源码
2. 安装 aarch64-linux-gnu- 交叉编译工具链
3. 编译 defconfig，生成内核 Image 和 dtb
4. 在 QEMU 中运行（用 virt 机器模型，先用通用的，再换高通的）
```

### 第2步：阅读设备树（2-3周）
```
arch/arm64/boot/dts/qcom/
  先读 msm8916.dtsi    — 简单，注释多
  再读 sdm845.dtsi     — 你的目标平台
  最后读 msm8998.dtsi  — 复杂但文档完善
```

### 第3步：从简单驱动入手（4-6周）
```
1. GPIO驱动：drivers/pinctrl/qcom/pinctrl-msm.c
   理解 TLMM (Top Level Mode Multiplexer)
2. 时钟驱动：drivers/clk/qcom/gcc-msm8998.c
   理解 RCG/CBCR 的注册和操作
3. 写一个简单的 platform driver 来操作 GPIO
```

### 第4步：深入 SMMU（8-12周）
```
1. 理解 IOMMU 框架：drivers/iommu/iommu.c
2. 阅读 arm-smmu.c：初始化、attach/detach、map/unmap
3. 研究 arm-smmu-qcom.c 的高通特定实现
4. 学习 DPD (Deep Power Down) 相关的电源管理代码
5. 用 QEMU 模拟 SMMU（arm-smmu-v3 在 QEMU 中有实现）
```

### 第5步：完整平台开发（12周+）
```
1. 在真机 DragonBoard 845c 上调试驱动
2. 用 ftrace 追踪 SMMU map/unmap 调用
3. 分析 ramdump，定位 SMMU fault
4. 参与 linux-arm-msm 邮件列表讨论
```
