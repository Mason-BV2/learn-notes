## 三、C 语言的正确学习顺序（零基础版）

```
────────── 第1-2周：环境 + 基础语法 ──────────
  安装 GCC 编译器
  理解 main() 函数
  变量类型（int / char / float / double）
  输入输出（printf / scanf）
  运算符（+ - * / % ++ --）
  流程控制（if / else / switch / for / while）

  📁 练习：写一个计算器（加减乘除）

────────── 第3-4周：数组 + 函数 ──────────
  一维数组、二维数组
  函数定义与调用
  函数参数传递（值传递 vs 地址传递）
  作用域与生命周期
  递归

  📁 练习：写一个冒泡排序 + 二分查找

────────── 第5-7周：指针（核心难点！）──────────
  ★ 这是 C 语言最核心的概念，多花时间
  指针是什么（内存地址）
  & 取地址 / * 解引用
  指针与数组的关系
  指针与函数（函数指针）
  多级指针
  malloc / free / calloc / realloc
  栈 vs 堆内存分配

  📁 练习：写一个动态数组（可自动扩容）

────────── 第8-9周：结构体 + 文件操作 ──────────
  结构体（struct）定义与使用
  枚举（enum）与联合体（union）
  文件读写（fopen / fread / fwrite / fclose）
  二进制文件 vs 文本文件

  📁 练习：写一个学生信息管理系统（增删改查）

────────── 第10-12周：进阶 C + 综合项目 ──────────
  volatile（内核开发必用！）
  位运算（~ & | ^ << >>）（寄存器操作必用）
  static 与 extern
  const 与指针
  函数指针与回调
  宏的高级用法（#、##、do-while(0)）
  链表、栈、队列的实现

  📁 练习1：写一个简单 Shell（支持管道和重定向）
  📁 练习2：写一个内存池分配器
```

## 四、内核开发中 C 语言的特殊用法

作为未来的内核工程师，在学完基础 C 之后，需要特别关注：

| C 语言特性 | 在内核中的用途 |
|------------|---------------|
| **位运算** | 寄存器操作、标志位设置、内存对齐 |
| **volatile** | 防止编译器优化 MMIO 和共享变量 |
| **container_of 宏** | 从结构体成员指针反推结构体基址 |
| **likely/unlikely** | 分支预测优化 |
| **WRITE_ONCE/READ_ONCE** | 防止编译器乱序 |
| **__init / __exit** | 标记初始化/清理函数放在特殊段 |
| **链表(list_head)** | 内核到处使用的双向链表 |
| **atomic_t** | 原子操作类型 |
| **内存屏障** | smp_mb() / smp_wmb() 等 |

## 六、C 语言学习避坑指南

1. **指针是必须攻克的** — 很多人在这里放弃了，别成为其中之一
2. **不要用 IDE 自动补全学习** — 手写代码，理解每个字符的含义
3. **理解内存模型** — 画内存图！知道每个变量在栈上还是堆上
4. **位运算要熟练** — 内核驱动开发几乎天天用
5. **不要跳过文件操作** — 虽然内核不用 fopen，但理解文件抽象很重要
6. **volatile 很关键** — 不是性能优化关键字，是正确性关键字
