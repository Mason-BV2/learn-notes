## 二、与你现有学习计划的关系

你的项目已经规划了**四阶段**学习计划，和上面的 CS 自学路线可以这样对应：

```
你的阶段一（计算机基础）
  └── 对应 CS 路线的：Crash Course → CS50 → CS61A 前半

你的阶段二（Linux基础）
  └── 对应 CS 路线的：Linux 命令行 + Shell 脚本 + 系统编程

你的阶段三（Linux内核基础）
  └── 对应 CS 路线的：CS61C（体系结构）+ CS162（操作系统）+ 自研内核

你的阶段四（高通平台专项）
  └── 超出 CS 路线范围，进入专业领域
```

## 三、你最需要的 5 门课（按优先级）

| 优先级 | 课程 | 为什么对高通内核工程师重要 |
|--------|------|--------------------------|
| 1 | **CS50 计算机科学导论** | 零基础入门，建立编程思维 |
| 2 | **自学 C 语言** | 内核的母语，不学走不动路 |
| 3 | **CS61C 体系结构** | 理解 CPU/内存/总线，内核开发天天接触 |
| 4 | **CS162 操作系统** | 内核本身就是操作系统，不理解OS没法写内核 |
| 5 | **MIT 6.S081 (xv6)** | 读一个小操作系统源码，理解内核各子系统 |

## 四、零基础起步第一周

```
Day 1-2: 了解计算机是什么
  → B站搜索 Crash Course Computer Science
  → 看前5集（计算机早期历史、电子计算、布尔逻辑、二进制、算术逻辑单元）
  → 建立整体认知

Day 3-4: 学习计算机组成原理（你的01.md课程）
  → 阅读你已有的计算机组成原理课程
  → 完成第1课练习

Day 5-7: 开始学 C 语言
  → 安装 GCC 编译器
  → 写出第一个 printf("Hello World\n");
  → 理解变量、输入输出、if/else
  → 看 翁恺 C语言 第一周视频

每日习惯：
  → 至少写30行 C 代码
  → 把练习结果记录到这个 APK 中
  → 遇到不懂的概念，去 小林coding 搜索图解
```

## 五、学习心态建议

1. **这是马拉松，不是短跑** — 3-5 年的路，不要急
2. **一致性 > 强度** — 每天30分钟好过周末搞12小时
3. **输出倒逼输入** — 把学的内容写成笔记/博客/视频
4. **接受不懂** — 内核开发没有"全懂"的人，保持好奇心
5. **这个 APK 就是你的知识库** — 持续维护，比教科书更实用
