# 计算机系统概论

## 目标

本课程预计需要10-15分钟时间，介绍计算机系统的基本组成和概念，为后续深入学习计算机组成原理打下坚实基础。

## 核心问题

**什么是计算机系统？计算机系统由哪些部分组成？各部分之间的关系是什么？**

计算机系统是现代信息技术的核心，理解其基本原理对于学习Linux内核开发、进行Qualcomm平台底层开发至关重要。正如你在SMMU DPD Proxy调试工作中所体验到的，所有的软件操作最终都涉及对硬件的控制和管理。

## 第一性原理/核心模型

### 计算机系统的层次结构

计算机系统由硬件和软件两大部分组成，它们相互依存，缺一不可。根据唐朔飞《计算机组成原理》（第2版）的定义，计算机系统的层次结构如下：

```
┌─────────────────────────────────────┐
│           应用软件层                 │
├─────────────────────────────────────┤
│           系统软件层                 │
│    （操作系统、编译程序等）           │
├─────────────────────────────────────┤
│           硬件层                     │
│  （控制器、运算器、存储器、I/O）       │
└─────────────────────────────────────┘
```

### 计算机硬件的基本组成

根据冯·诺依曼提出的"存储程序"原理，计算机硬件由五大部件组成：

1. **运算器**：负责进行算术运算和逻辑运算，是计算机的执行部件。在Qualcomm的Snapdragon处理器中，运算器采用ARM架构的ALU（算术逻辑单元）。

2. **控制器**：协调和控制计算机各部件执行指令，包括取指令、分析指令、执行指令等。控制器与运算器合称为**中央处理器（CPU）**。

3. **存储器**：存储程序和数据。在计算机术语中，"内存"通常指主存储器（RAM），"外存"指硬盘、SSD等。Qualcomm平台中，SMMU（系统内存管理单元）负责管理内存访问。

4. **输入设备**：将外部信息输入计算机，如键盘、鼠标、传感器等。

5. **输出设备**：将计算机处理结果输出，如显示器、打印机等。

### 计算机软件的分类

软件分为系统软件和应用软件两大类：

- **系统软件**：管理计算机系统资源，为应用软件提供公共服务。包括操作系统（如Linux）、编译程序、汇编程序、数据库管理系统等。

- **应用软件**：解决特定问题的软件，如办公软件、浏览器、生物信息分析软件等。

## 关键概念与例子

### 1. 冯·诺依曼体系结构

**参考教材**：唐朔飞《计算机组成原理》（第2版），第1章第3节

冯·诺依曼体系结构的核心思想是"存储程序"（Stored-program），其特点包括：

1. 计算机（硬件）由五大部件组成
2. 程序和数据以二进制形式存储在内存中
3. 计算机自动连续执行指令
4. 以运算器为中心

**考研真题示例**（2019年清华大学912真题）：
> 冯·诺依曼体系结构的核心思想是什么？它有哪些主要特点？

**你的工作中**：你在Qualcomm平台做的SMMU DPD Proxy调试，就是基于冯·诺依曼体系结构的实际应用——程序存储在内存中，CPU通过SMMU控制对内存的访问权限。

### 2. 计算机系统的主要性能指标

**参考教材**：唐朔飞《计算机组成原理》（第2版），第1章第5节

计算机系统的主要性能指标包括：

- **机器字长**：CPU一次能处理的二进制位数。常见有32位和64位。Qualcomm Snapdragon 8系列采用64位ARM架构。

- **运算速度**：用MIPS（Million Instructions Per Second，每秒执行百万条指令）或FLOPS（Floating-point Operations Per Second，每秒浮点运算次数）来衡量。

- **存储容量**：包括内存容量和外存容量。内存容量决定程序运行的临时存储空间大小。

- **带宽**：单位时间内传输的数据量，通常以MB/s或GB/s衡量。

### 3. 计算机的工作过程

计算机的工作过程就是执行指令的过程，基本分为以下步骤：

1. **取指令**：从内存取出指令到CPU
2. **分析指令**：控制器解析指令的操作码和地址码
3. **执行指令**：运算器完成相应操作
4. **存储结果**：将结果存回内存

**考研真题示例**（2020年清华大学912真题）：
> 简述计算机执行一条指令的基本过程。

### 4. 总线系统

**参考教材**：唐朔飞《计算机组成原理》（第2版），第6章

总线是连接计算机各部件的公共信息线，按功能分为三类：

- **数据总线**：传输数据，双向传输
- **地址总线**：传输内存地址，单向传输
- **控制总线**：传输控制信号

总线带宽 = 总线宽度 × 总线频率

**Qualcomm平台实例**：在Snapdragon处理器中，CPU、L2缓存、SMMU、DMA控制器等通过AXI总线互联，总线带宽直接影响系统性能。

## 常见坑

1. **混淆内存和外存**：内存（RAM）是易失性的，断电后数据丢失；外存（SSD、硬盘）是非易失性的。在Linux内核中，内存分配（kmalloc、vmalloc）和块设备读写操作完全不同。

2. **忽视总线瓶颈**：在设计数据传输时，总线带宽可能成为性能瓶颈。例如，在调试SMMU时，需要考虑地址总线和数据总线的负载。

3. **不理解存储程序原理**：很多人在学习时会忽视程序和数据都以二进制形式存储在内存中这一基本事实，导致对后续的指针、内存管理等概念理解困难。

4. **混淆CPU和GPU**：CPU是通用处理器，适合串行任务；GPU是专用处理器，适合并行任务。在Qualcomm的Snapdragon中，GPU主要负责图形处理，但也可以用于通用计算（GPGPU）。
