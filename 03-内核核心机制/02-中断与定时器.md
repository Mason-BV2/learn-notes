# 中断与定时器

## 概述

中断是硬件通知 CPU 事件发生的机制。定时器是基于时间触发操作的机制。这两者是内核驱动开发和高通平台调试的核心技能。

> 高通工作场景：调试外设中断不工作、IRQ affinity 设置、pinctrl 中断映射、高精度定时器在相机/音频驱动中的应用。

## 中断机制

### 基础概念

```
硬件设备 → IRQ → GIC（中断控制器）→ CPU → do_IRQ() → 中断处理函数
```

### 注册中断

```c
#include <linux/interrupt.h>

// 中断处理函数
static irqreturn_t qcom_irq_handler(int irq, void *dev_id)
{
    // 上半部（硬中断上下文）
    // 快速响应硬件，不能 sleep
    // 通常只做：读取状态寄存器、清除中断标志、调度下半部
    
    return IRQ_HANDLED;       // 已处理
    // return IRQ_WAKE_THREAD;  // 唤醒 threaded handler
}

// 注册中断
int request_irq(unsigned int irq, irq_handler_t handler,
                unsigned long flags, const char *name, void *dev);

// 带线程的注册方式
int request_threaded_irq(unsigned int irq,
                          irq_handler_t handler,       // 上半部（可 NULL）
                          irq_handler_t thread_fn,     // 下半部线程
                          unsigned long flags,
                          const char *name, void *dev);

// 示例：高通 I2C 中断
ret = request_threaded_irq(qcom_dev->irq,
                           qcom_i2c_irq_handler,  // 上半部：快速检查状态
                           qcom_i2c_irq_thread,    // 下半部：处理数据传输
                           IRQF_TRIGGER_HIGH | IRQF_ONESHOT,
                           "qcom_i2c", qcom_dev);
```

### 中断标志

```c
IRQF_TRIGGER_RISING     // 上升沿触发
IRQF_TRIGGER_FALLING    // 下降沿触发
IRQF_TRIGGER_HIGH       // 高电平触发
IRQF_TRIGGER_LOW        // 低电平触发
IRQF_SHARED             // 共享中断（多个设备共用一个 IRQ）
IRQF_ONESHOT            // 保持中断禁用直到 threaded handler 完成
IRQF_NO_SUSPEND         // 系统休眠时不禁用
```

### 上半部 vs 下半部

```
上半部 (hardirq)
  ├── 必须快速执行（~微秒级）
  ├── 不可 sleep（原子上下文）
  ├── 禁止本地中断
  └── 通常：ack 中断、保存寄存器、调度下半部

下半部
  ├── softirq       （软中断，内核预定义）
  ├── tasklet       （基于 softirq 的动态下半部）[已过时]
  ├── workqueue     （内核线程上下文，可 sleep）← 推荐
  └── threaded irq  （内核线程上下文）← 高通常用
```

### Threaded IRQ — 高通驱动的标准做法

```c
// 高通驱动中最常见的模式
// 上半部检查硬件状态，返回 IRQ_WAKE_THREAD
// thread_fn 在独立内核线程中执行，可以 sleep

static irqreturn_t qcom_geni_i2c_irq(int irq, void *dev)
{
    // 上半部：非常快，只检查状态
    writel_relaxed(0xFFFFFFFF, base + GENI_M_IRQ_CLEAR);
    
    // 需要更多处理？唤醒线程
    if (需要进一步处理)
        return IRQ_WAKE_THREAD;
    return IRQ_HANDLED;
}

static irqreturn_t qcom_geni_i2c_thread(int irq, void *dev)
{
    // 下半部：可以做任何操作，包括 sleep
    // I2C 传输、DMA 操作、等待硬件
    return IRQ_HANDLED;
}
```

## GIC — 通用中断控制器

ARM 平台使用 GIC（Generic Interrupt Controller）：

```
GIC 架构
  GICD  →  Distributor（分发器）：中断优先级、使能、路由
  GICR  →  Redistributor（重分发器）：每个 CPU 一个
  ICC   →  CPU Interface（CPU 接口）：CPU 侧寄存器

中断类型
  SGI   → Software Generated Interrupt（核间通信 IPI）
  PPI   → Private Peripheral Interrupt（私有外设中断，如 arch timer）
  SPI   → Shared Peripheral Interrupt（共享外设中断，如 I2C、UART）
  LPI   → Locality-specific Peripheral Interrupt（新型，MSI 相关）
```

```bash
# 查看系统中的中断
cat /proc/interrupts

# 查看 GIC 信息
cat /proc/irq/*/chip_name  # 显示中断控制器类型

# 设置 IRQ affinity（绑定到特定 CPU）
echo 2 > /proc/irq/123/smp_affinity  # CPU1 处理该中断
```

## 定时器

### 低精度定时器 — timer_list

```c
#include <linux/timer.h>

struct timer_list qcom_timer;

void qcom_timer_callback(struct timer_list *t)
{
    pr_info("Timer expired!\n");
    // 如果需要重复触发，重新添加
    mod_timer(&qcom_timer, jiffies + msecs_to_jiffies(100));
}

void init_function(void)
{
    timer_setup(&qcom_timer, qcom_timer_callback, 0);
    mod_timer(&qcom_timer, jiffies + msecs_to_jiffies(100));
    // 100ms 后触发
}

void cleanup_function(void)
{
    del_timer_sync(&qcom_timer);  // 安全删除（等待正在运行的 handler 完成）
}
```

### 高精度定时器 — hrtimer

```c
#include <linux/hrtimer.h>
#include <linux/ktime.h>

struct hrtimer qcom_hrtimer;

static enum hrtimer_restart qcom_hrtimer_cb(struct hrtimer *timer)
{
    // 在 hrtimer 上下文中执行
    // 精度可达纳秒级（硬件支持的话）
    
    // 重复触发
    hrtimer_forward_now(timer, ns_to_ktime(1000000));  // 1ms
    // 或单次触发
    // return HRTIMER_NORESTART;
    
    return HRTIMER_RESTART;
}

void init_hrtimer(void)
{
    hrtimer_init(&qcom_hrtimer, CLOCK_MONOTONIC, HRTIMER_MODE_REL);
    qcom_hrtimer.function = qcom_hrtimer_cb;
    hrtimer_start(&qcom_hrtimer, ns_to_ktime(1000000), HRTIMER_MODE_REL);
    // 1ms 后触发
}

void cleanup_hrtimer(void)
{
    hrtimer_cancel(&qcom_hrtimer);
}
```

### jiffies — 内核时间的脉搏

```c
// jiffies = 自系统启动以来的 tick 数
// HZ = 每秒 tick 数（ARM64 通常 250 或 1000）

// 时间转换
unsigned long j = jiffies;
msecs_to_jiffies(100);      // 100ms → jiffies
jiffies_to_msecs(100);      // jiffies → ms
jiffies_to_usecs(100);      // jiffies → us

// 时间比较
time_after(jiffies, timeout);    // 是否超时
time_before(jiffies, deadline);  // 是否在截止前
```

### clockevents / clocksource

```
硬件 Timer → clockevents → tick device → jiffies 更新
                                 → hrtimer 触发
                                 → softirq (TIMER_SOFTIRQ)

ARM Generic Timer（通用定时器）
  CNTPCT → Physical Count Register（物理计数器）
  CNTP_CVAL → Compare Value（比较值）  
  CNTP_TVAL → Timer Value（定时器值）
  CNTFRQ  → Timer Frequency（定时器频率）
```

```bash
# 查看系统中的 clocksource
cat /sys/devices/system/clocksource/clocksource0/current_clocksource
# ARM 架构通常输出：arch_sys_counter

# 查看可用 clocksource
cat /sys/devices/system/clocksource/clocksource0/available_clocksource
```

## 面试考点

1. **中断上下文中不能做什么？** 不能 sleep、不能获取 mutex、不能访问用户空间、不能做页错误处理
2. **中断上半部和下半部怎么分工？** 上半部处理时间敏感的操作（ACK、保存寄存器），下半部处理剩余操作
3. **request_threaded_irq 的优势？** 减少内核栈压力、线程可 sleep、与实时内核兼容
4. **GIC 中 SPI 和 PPI 的区别？** SPI 可以被任何一个 CPU 处理（共享外设），PPI 只能被指定的 CPU 处理（私有外设）
5. **jiffies 回绕问题怎么处理？** 使用 time_after/time_before 宏（无符号整数回绕是安全的）
