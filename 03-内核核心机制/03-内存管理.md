# 内存管理

## 概述

Linux 内核内存管理负责物理内存分配、虚拟地址映射、页表管理、SLAB 分配器和 DMA 缓冲区管理。在高通平台上，ION 内存分配器和 CMA 是大块连续内存分配的关键。

> 高通工作场景：ION 分配器用于 camera/display 的内存分配、DMA 缓冲区管理、SMMU/IOMMU 配置、系统内存压力下的 OOM 调试。

## 内核地址空间

```
ARM64 内核地址空间布局（48-bit）：

0x0000_0000_0000_0000  →  用户空间 (0 - 2^47)
0x0000_ffff_ffff_ffff
0xffff_0000_0000_0000  →  内核空间 (2^47 - 2^48)
0xffff_ffff_ffff_ffff

内核段分布（从低到高）：
  vmalloc 区     → vmalloc()
  vmemmap        → struct page 数组
  PCI I/O 区
  modules        → 模块加载区
  vmalloc
  Fixmap
  FIXADDR_START
  PCI I/O
  vmemmap
  linear mapping → 直接映射区（virt → phys 直接转换）
```

### 直接映射区（Linear Mapping）

```c
// ARM64 上物理内存直接映射到内核地址空间
// virt_to_phys / phys_to_virt 直接转换

// virt_addr = phys_addr + PAGE_OFFSET - PHYS_OFFSET
// 或使用宏：
void *virt = phys_to_virt(phys_addr);
phys_addr_t phys = virt_to_phys(virt_addr);
```

## 物理内存管理

### Buddy System — 物理页分配器

```c
// 以 2^n 个连续 page 为单位的分配器
struct page *alloc_pages(gfp_t gfp_mask, unsigned int order);
// order=0 → 1 page, order=1 → 2 pages, ..., order=10 → 1024 pages

void free_pages(unsigned long addr, unsigned int order);

// 常用 GFP 标志
GFP_KERNEL    // 内核分配（可能休眠）
GFP_ATOMIC    // 原子分配（不可休眠）
GFP_DMA       // DMA 可用内存（低 16MB 或 32-bit）
GFP_HIGHUSER  // 用户空间页面

// 检查内存碎片
cat /proc/buddyinfo
```

### SLAB/SLUB 分配器 — 小对象分配

```c
// SLUB 是当前默认的 slab 分配器
// 管理大小固定的对象缓存（如 task_struct、inode）

// 创建缓存
struct kmem_cache *cache = kmem_cache_create("qcom_cache",
                                               sizeof(struct qcom_obj),
                                               0, 0, NULL);

// 分配对象
struct qcom_obj *obj = kmem_cache_alloc(cache, GFP_KERNEL);

// 释放
kmem_cache_free(cache, obj);

// 销毁缓存
kmem_cache_destroy(cache);

// 查看 slab 使用情况
cat /proc/slabinfo | grep qcom
```

### vmalloc — 虚拟地址连续分配

```c
// 物理地址可以不连续，虚拟地址连续
// 适合大块分配（但性能不如 kmalloc）
void *buf = vmalloc(1024 * 1024);  // 1MB
// ... 使用 ...
vfree(buf);
```

## 页表与 MMU

ARM64 支持 4K/16K/64K 页大小，页表级数 3-4 级：

```
                    ARM64 4K 页 4级页表
                    
TTBR0_EL1 (用户)     L0 (PGD) → L1 (PUD) → L2 (PMD) → L3 (PTE) → 4K 页
TTBR1_EL1 (内核)     L0 (PGD) → L1 (PUD) → L2 (PMD) → L3 (PTE) → 4K 页
```

```c
// 页表相关操作
pgd_t *pgd = pgd_offset(mm, addr);
pud_t *pud = pud_alloc(mm, pgd, addr);
pmd_t *pmd = pmd_alloc(mm, pud, addr);
pte_t *pte = pte_alloc_map(mm, pmd, addr);

// 设置页表项
set_pte(pte, pfn_pte(pfn, prot));
```

## 连续内存分配器 — CMA

CMA 为大块连续内存需求而设计（camera、display、video 编解码）：

```c
// DTS 中定义 CMA 区域
// arch/arm64/boot/dts/qcom/sm8250.dtsi
reserved-memory {
    #address-cells = <2>;
    #size-cells = <2>;
    ranges;
    
    cont_splash_mem: cont-splash-region@9d800000 {
        reg = <0x0 0x9d800000 0x0 0x2300000>;
    };
    
    cma_mem: linux,cma {
        compatible = "shared-dma-pool";
        reusable;
        alloc-ranges = <0x0 0x00000000 0x0 0xffffffff>;
        size = <0x0 0x20000000>;  // 512MB CMA
    };
};

// 内核中使用 CMA
#include <linux/cma.h>

struct page *cma_alloc(struct cma *cma, unsigned long count,
                       unsigned int align, bool no_warn);

// 当前系统 CMA 信息
cat /proc/meminfo | grep Cma
```

## DMA 缓冲区管理

```c
#include <linux/dma-mapping.h>

// DMA 地址和物理地址不同（IOMMU/SMMU 映射后）
// 驱动应该始终使用 DMA API，而不是直接操作物理地址

// 分配 DMA 缓冲区
dma_addr_t dma_handle;
void *cpu_addr = dma_alloc_coherent(dev, size, &dma_handle, GFP_KERNEL);
// cpu_addr  → CPU 访问的虚拟地址
// dma_handle → 设备访问的 DMA 地址（已经过 IOMMU 映射）

// 流式 DMA 映射（用于网络包等一次性传输）
dma_addr_t dma_addr = dma_map_single(dev, cpu_addr, size, DMA_TO_DEVICE);
dma_unmap_single(dev, dma_addr, size, DMA_TO_DEVICE);

// 同步
dma_sync_single_for_cpu(dev, dma_addr, size, DMA_FROM_DEVICE);
dma_sync_single_for_device(dev, dma_addr, size, DMA_FROM_DEVICE);
```

### 高通 ION 内存分配器

ION 是高通平台专用的内存分配器，管理多个不同的内存池（heap）：

```c
// ION heap 类型（高通平台）
enum ion_heap_type {
    ION_HEAP_TYPE_SYSTEM,       // 系统内存（内核 buddy）
    ION_HEAP_TYPE_SYSTEM_CONTIG, // 连续系统内存
    ION_HEAP_TYPE_CARVEOUT,     // 预留内存区域
    ION_HEAP_TYPE_DMA,          // DMA 内存池
    ION_HEAP_TYPE_CUSTOM,       // 自定义（如 qcom 的 secure heap）
};

// DTS 中定义 ION heap
qcom,ion {
    compatible = "qcom,ion";
    
    system-heap {
        memory-region = <&system_cma>;
        qcom,ion-heap-id = <0>;
        qcom,ion-heap-type = "system";
    };
    
    qcom,adsp-heap {
        memory-region = <&adsp_mem>;
        qcom,ion-heap-id = <22>;
        qcom,ion-heap-type = "carveout";
    };
};
```

### 2. 使用 mmap 实现零拷贝驱动

```c
// 驱动端
static int qcom_mmap(struct file *filp, struct vm_area_struct *vma)
{
    struct qcom_device *dev = filp->private_data;
    unsigned long pfn = virt_to_phys(dev->buffer) >> PAGE_SHIFT;
    
    return remap_pfn_range(vma, vma->vm_start, pfn,
                           vma->vm_end - vma->vm_start,
                           vma->vm_page_prot);
}
```

## 面试考点

1. **kmalloc 和 vmalloc 的区别？** kmalloc 保证物理连续（适合 DMA），vmalloc 只保证虚拟连续（适合大块分配）
2. **什么是匿名页？** 无文件映射的页面（堆、栈、写时复制），swap 时写回 swap 分区
3. **什么是 mmap 零拷贝？** 用户态直接映射内核/设备内存，避免 read/write 时的数据拷贝
4. **CMA 的工作机制？** CMA 区域在未被驱动使用时，可以被应用程序作为可移动页面使用；当驱动请求 CMA 时，内核会迁移页面腾出空间
5. **OOM Killer 如何选择进程？** 根据 badness score（内存占用、进程优先级、运行时间等）
