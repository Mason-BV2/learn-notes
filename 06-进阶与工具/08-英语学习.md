# 🌎 高通英语学习指南 — 内核工程师专项

## 概述

高通是美资外企，英语是日常工作语言。你需要：
- **读**：芯片 datasheet、内核邮件列表、技术文档、commit message
- **写**：patch 提交、review 回复、技术邮件、设计文档
- **听**：技术会议、代码 review、客户 call、面试
- **说**：每日站会、技术讨论、面试问答

> 这不是通用英语学习，是**专为高通内核工程师定制的技术英语**。

## 二、阅读训练（Reading）

### 2.1 必读英文资料

**每天必读**（15分钟）：
```bash
# 1. Linux kernel mailing list (LKML)
# https://lore.kernel.org/lkml/
# 每天看 3-5 个 patch 的标题和描述

# 2. Qualcomm 相关 patch
# https://lore.kernel.org/linux-arm-msm/
# 关注 qcom 开头的 patch

# 3. 英文技术文章
# https://lwn.net/  (Linux Weekly News)
# https://www.kernel.org/doc/html/latest/
```

### 2.2 阅读方法

```
Step 1: 扫读标题（~1分钟）
   理解 patch 是修 bug / 加功能 / 重构

Step 2: 读 commit message（~3分钟）
   看懂 "Why" — 为什么需要这个修改

Step 3: 看 diff（~5分钟）
   理解 "What" — 具体修改了什么

Step 4: 记词汇（~1分钟）
   收集 3-5 个不认识的词
```

**实战练习**：每周精读 3 个内核 patch

```text
Example — 真实的内核 commit message:

commit 3c8fb5048b6b1234abcd1234efgh5678ijkl9012
Author: John Doe <john.qualcomm.com>
Date:   Mon Jan 15 14:30:00 2024 +0800

    i2c: qcom-geni: add support for I2C Master Hub serial engine
    
    I2C Master Hub is a new serial engine IP which is derived from
    existing GENI IP. It supports DMA mode transfer only and has a
    different interrupt path compared to the legacy GENI.
    
    Add support for this new variant by extending the existing
    qcom-geni I2C driver with new compatible and operations.
    
    Signed-off-by: John Doe <john@qualcomm.com>

→ 学习点：
  "derived from" = 源自
  "legacy" = 旧的/遗留的  
  "variant" = 变体
  "extending" = 扩展
  "compatible" = 兼容的（DT compatible string）
```

## 三、写作训练（Writing）

### 3.1 Commit Message 写作

**模板** (\<module\>: \<description\>):

```
<module>: <Brief description (max 70 chars)>

<Detailed explanation of what and why.
Explain the problem being solved.
Line wrap at 72 characters.>

Signed-off-by: Your Name <your.email@example.com>
```

**练习**：把你的项目写成英文 commit message

```text
练习：写一个字符设备驱动的 commit message

→ 答案示例：
char: qcom_char: add character device driver example

Add a simple character device driver for learning purposes.
The driver supports open, read, write, release and ioctl
operations. It uses mutex for concurrent access protection
and udev for automatic device node creation.

This driver serves as a reference for Qualcomm platform
driver development.

Signed-off-by: Zhang San <zhangsan@example.com>
```

### 3.2 技术邮件写作

```text
Subject: [PATCH v2 1/3] drivers: qcom: add SMMU support

Hi Bjorn,

This patch adds SMMU support for the Qualcomm SM8250
platform. The implementation follows the ARM SMMUv3
specification.

Changes since v1:
- Fixed DMA mapping issue (suggested by Will Deacon)
- Added proper error handling in probe function
- Updated DT binding documentation

Please review.

Thanks,
Zhang San
```

### 3.3 英文技术写作模板

| 场景 | 常用句式 |
|------|---------|
| 报告 Bug | `I observed an issue where ...` / `The system crashes when ...` |
| 提出方案 | `I propose to ...` / `The approach is to ...` |
| 请求 Review | `Please review the patch at ...` / `Could you take a look?` |
| 回复意见 | `Thanks for the feedback. I'll fix it in v2.` / `I agree with your suggestion.` |
| 同步进度 | `Currently I'm working on ...` / `The progress is blocked by ...` |
| 询问问题 | `I have a question about ...` / `Could you clarify ...?` |

### 4.2 听力训练方法

```
精听法（15分钟/天）：
Step 1: 不看字幕听一遍（理解大意）
Step 2: 看字幕/原文听一遍（查生词）
Step 3: 跟读一遍（模仿发音和语速）
Step 4: 整理 3-5 个新表达

推荐视频主题（直接搜索 YouTube）：
- "Linaro connect KVM" — 虚拟化相关
- "Linux kernel scheduler" — 调度器
- "Device tree for beginners" — 设备树
- "Qualcomm embedded Linux" — 高通平台
```

### 4.3 技术会议常用句式

```text
Meeting phrases:

Opening:
  "Let's start with the status update."
  "Today's agenda covers three topics."

Asking for status:
  "What's the status of the SMMU driver?"
  "Are there any blockers?"

Giving status:
  "I've completed the I2C driver bring-up."
  "I'm currently working on the DMA integration."
  "I'm blocked by the toolchain issue."

Technical discussion:
  "Have you considered using threaded IRQ instead?"
  "The root cause might be in the clock driver."
  "We need to backport this fix to android-5.10."
```

## 五、口语训练（Speaking）

### 5.1 技术面试口语

**高频面试问题 — 英文回答练习**：

```text
Q1: "Tell me about yourself."
A1: "I have X years of experience in Linux kernel development, 
     focusing on ARM64 platform drivers. I've worked on device 
     tree, I2C/SPI drivers, and kernel debugging. Currently 
     I'm preparing for a kernel engineer role at Qualcomm."

Q2: "Describe a difficult bug you solved."
A2: "I encountered a kernel panic during platform bring-up. 
     The system crashed immediately after enabling the I2C 
     controller. I used ftrace to trace the function calls and 
     found that the clock was not properly configured. The root 
     cause was a missing clock entry in the device tree. I added 
     the clock phandle and the issue was resolved."

Q3: "What's your experience with ARM architecture?"
A3: "I have working knowledge of ARMv8-A architecture, 
     including exception levels, MMU with stage-2 translation, 
     GICv3 interrupt controller, and cache coherency. I've 
     written low-level assembly code for CPU bring-up and 
     configured page tables for KVM virtualization."

Q4: "Why do you want to work at Qualcomm?"
A4: "Qualcomm is a leader in mobile SoCs and I'm passionate 
     about Linux kernel development on ARM platforms. I want 
     to contribute to cutting-edge kernel technologies like 
     pKVM, EAS scheduler, and platform power management."
```

### 5.2 每日口语练习

```bash
# 方法：Shadowing（影子跟读法）
# 每天 10 分钟

# Step 1: 找一段 2 分钟的英文技术演讲
# Step 2: 播放 -> 延迟 0.5 秒跟读
# Step 3: 模仿发音、语调、停顿
# Step 4: 录音 -> 回听 -> 对比

# 推荐素材：
# YouTube: "LWN: Kernel development in 2024"
# YouTube: "Linaro Connect - KVM on ARM"
```

### 5.3 外企每日场景英语

```text
场景1：每日晨会 (Daily Standup)
  "Good morning everyone. Yesterday I worked on the I2C 
   driver probe issue. Today I'll continue debugging the 
   DMA transfer problem. No blockers."

场景2：代码 Review
  "Thanks for the patch. I have a few comments:
   1. The error handling in probe() should use goto pattern.
   2. Please add the MODULE_DEVICE_TABLE for OF matching.
   3. The spinlock should be initialized before request_irq."

场景3：向老板汇报
  "Hi Manager, I'd like to give you a quick update on 
   the bring-up status. The kernel boots successfully 
   on the new platform, but the eMMC driver is still 
   not working. I estimate it will take 2 more days."

场景4：跨团队沟通
  "Hi Hardware Team, could you clarify the interrupt 
   mapping for the I2C controller? The DT binding shows 
   IRQ 123 but the hardware spec says IRQ 127."
```

```
周进度：
```

## 八、面试英语专项

### 面试自我介绍模板

```text
Self-Introduction (90 seconds):

"Good morning/afternoon. My name is Zhang San. 
I have been working in Linux kernel development for 
X years, focusing on ARM64 platform drivers.

In my previous role, I was responsible for bringing up 
new Qualcomm platforms. I developed device drivers for 
I2C, SPI, and UART peripherals using the Linux kernel 
driver framework. I have experience with device tree, 
kernel debugging, and performance optimization.

I'm particularly interested in the Qualcomm kernel 
engineer position because I'm passionate about ARM 
architecture and I want to contribute to the cutting-edge 
kernel technologies at Qualcomm.

Thank you for this opportunity. I look forward to 
discussing how my experience aligns with your needs."
```

### 高频英文面试题

```text
1. "Describe your experience with Linux kernel development."
   重点：谈具体项目、用什么 API、解决过什么问题

2. "Explain how a character device driver works."
   重点：file_operations / cdev / probe / ioctl

3. "How do you debug a kernel panic?"
   重点：printk → ftrace → kgdb → kdump 的递进方法

4. "What's your experience with ARM architecture?"
   重点：异常级别 / MMU / GIC / 缓存一致性

5. "Describe a time you had a conflict with a colleague."
   重点：STAR 法则 (Situation/Task/Action/Result)

6. "How do you stay updated with kernel development?"
   重点：读 LKML / LWN / 参加 Linaro Connect

7. "Why do you want to work at Qualcomm?"
   重点：技术 passion / ARM / 内核 / 移动平台

8. "Where do you see yourself in 5 years?"
   重点：成为 ARM 内核专家 / 贡献上游
```
