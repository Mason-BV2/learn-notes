### 1. 链表反转

```c
// 面试题：反转单链表
struct list_node {
    int data;
    struct list_node *next;
};

struct list_node *reverse(struct list_node *head)
{
    struct list_node *prev = NULL;
    struct list_node *curr = head;
    struct list_node *next;
    
    while (curr) {
        next = curr->next;
        curr->next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}
```

### 4. 内存屏障

```c
// 面试题：为什么驱动中需要内存屏障？
// ARM 是弱内存序架构，CPU 可能重排内存访问

// 场景：写 MMIO 寄存器后立即读状态
writel(CMD_START, dev->base + REG_CMD);
// 如果没有屏障，读可能在写之前执行
status = readl(dev->base + REG_STATUS);

// 正确写法
writel(CMD_START, dev->base + REG_CMD);
wmb();  // 写内存屏障，确保之前的写完成
status = readl(dev->base + REG_STATUS);
```

```c
// 面试题：进程和线程的区别？
// 进程：独立地址空间，资源隔离，切换开销大
```

## 三、Linux 内核专题

### 1. 中断上下文

```c
// 面试题：为什么中断处理函数不能 sleep？
// - 中断上下文没有进程关联（current 指向被中断的进程）
// - 中断关闭了调度器
// - 在中断中 sleep 会导致系统 Hang

// 解决：把需要 sleep 的操作放到 threaded irq
```

### 2. spinlock vs mutex

```c
// 面试题：spinlock 和 mutex 的区别？
// spinlock：忙等待，不可睡眠，用于中断上下文
// mutex：可睡眠，用于进程上下文

// 规则：
// spinlock → 上下文：中断、tasklet、临界区很短
// mutex → 上下文：进程、可睡眠、临界区较长
```

### 3. kmalloc 和 vmalloc 区别

```c
// kmalloc: 物理连续，适合 DMA，但可能分配失败（碎片）
// vmalloc: 虚拟连续，物理可以不连续，适合大块分配
```

### 4. 内核态和用户态切换

```c
// 面试题：用户态调用 read() 到内核驱动的完整流程？
// 1. 用户态 read() → libc 封装
// 2. SVC 指令 → 切换到 EL1
// 3. 系统调用处理 (sys_read)
// 4. VFS 层 (vfs_read)
// 5. 具体文件系统/设备驱动 (file_operations.read)
// 6. copy_to_user 传回数据
// 7. 返回用户态
```

## 六、系统设计题

```c
// 设计一个高性能字符设备驱动，支持多进程并发访问

// 需要考虑：
// 1. 并发：mutex/semaphore 保护共享数据
// 2. 性能：使用 DMA + mmap 零拷贝
// 3. 阻塞：支持 O_NONBLOCK 和 poll
// 4. 多实例：每个 open 创建一个实例
// 5. 内存：使用 kfifo 或 ring buffer

// poll 接口
static unsigned int qcom_poll(struct file *filp, struct poll_table_struct *wait)
{
    struct qcom_dev *dev = filp->private_data;
    unsigned int mask = 0;
    
    poll_wait(filp, &dev->wq, wait);
    
    if (dev->data_ready)
        mask |= POLLIN | POLLRDNORM;
    
    return mask;
}
```
